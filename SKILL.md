# 视觉风格选择

渲染前询问用户希望视频是什么风格。

这是渲染前阻断步骤，不要直接进入渲染。

必须先读取 `references/video-archetypes.md` 获取每种预设的完整规范（配色、字体、动画参数、音乐建议），然后采用一问一答确认用户需求。

## 核心规则

- 不设置默认风格。
- 不因为内容是数据/新闻就自动选择某一种风格。
- 不因为用户说“随便”“默认”“你看着办”“直接渲染”就自动套用配置。
- 用户没有明确选择或描述画面需求时，必须继续追问。
- 一次只问一个关键问题，避免把用户淹没在选项里。

## 一问一答顺序

第一问必须确认视觉方向：

```text
这条视频你希望是什么画面感觉？可以直接选一个，也可以用自己的话描述。
```

第二问再确认视频比例：

```text
视频比例用 16:9 还是 9:16？
```

如果用户回答含糊，例如“默认”“随便”“你看着办”，不要继续渲染，改为追问：

```text
我不能直接套默认风格。你更想要哪种感觉：数据感、发布会感、故事感，还是技术感？
```

## 候选风格库（4种）

给用户展示，用中文解释画面感：

```
1. 暗海军蓝数据风（Data-Rich / Bloomberg）
   → 暗海军蓝底、动画图表、快速错落入场、数字计数器
   → 适合：数据报告、新闻、资讯、财报

2. 黑底发布极简风（Minimal / Jobs）
   → 纯黑底、巨大标题(80-96pt)、一帧一个想法、戏剧性停顿
   → 适合：产品发布、重要声明、概念表达

3. 暖色叙事风（Warm / Narrative）
   → 暖色调、衬线字体引用、柔和动效、人物照片
   → 适合：课程介绍、社区故事、人文内容

4. 终端技术风（Terminal / Hacker）
   → GitHub暗底、全等宽字体、打字机效果、逐行输出
   → 适合：技术演示、开发工具、CLI产品
```

## 引导方式

不要讲代码。用中文解释画面感。

如果用户不选，不生成 `runtime/style-profile.json`。

如果用户说"直接渲染""随便""默认"，不跳过本步骤，也不套默认；必须继续追问风格需求和比例。

## 选择后

1. 确认用户选择
2. 根据 `references/video-archetypes.md` 中对应预设的完整配色板、字体栈、动画参数，生成 `runtime/style-profile.json`
3. 交给 `animation-preset-builder` 继续

确认后必须输出给 `animation-preset-builder` 的风格结论，不能只停留在口头建议。

这是场景四第 7 步。确认风格后必须继续进入第 8 步 `animation-preset-builder`，生成 `runtime/style-profile.json`，不能直接渲染。

## 额外能力

如果用户提供了参考视频、截图或视觉风格链接，分析其配色、字体、动画速度和整体氛围，尝试匹配到最近的预设，或基于参考创建一个自定义风格配置。

## 下一步

把用户选择交给 `animation-preset-builder`。
