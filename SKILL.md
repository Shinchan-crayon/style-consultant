# 视觉风格选择

渲染前询问用户希望视频是什么风格。

这是渲染前阻断步骤，不要直接进入渲染。

必须向用户询问：

```text
这条视频准备做成什么视觉风格？默认推荐：浅色财经 MG。
```

如果用户不回复或明确说默认，选择 `浅色财经 MG`。

## 推荐默认

```text
浅色财经 MG
```

## 可选风格

给用户展示简单选项：

```text
1. 浅色财经 MG
2. 知识解释风
3. 新闻评论风
4. 强节奏短视频风
5. 深色数据分析风
```

## 引导方式

不要讲代码。用中文解释画面感：

- 浅色财经 MG：清爽、专业、卡片、数字、关键词高亮。
- 知识解释风：适合讲概念，结构更慢，层层展开。
- 新闻评论风：适合观点表达，标题和结论更有力量。
- 强节奏短视频风：节奏更快，文字冲击更强。
- 深色数据分析风：适合科技、链上、数据分析类内容，画面更冷静。

如果用户不选，使用默认风格。

确认后必须输出给 `animation-preset-builder` 的风格结论，不能只停留在口头建议。

这是场景四第 7 步。确认风格后必须继续进入第 8 步 `animation-preset-builder`，生成 `runtime/style-profile.json`，不能直接渲染。
